# Sci-Copilot 整改执行文档

本文档用于约束后续改动顺序：先文档、再实施、再验证。

## 目标

按优先级完成以下五项整改：

1. 处理密钥泄露风险
2. 清理并收敛核心服务代码（`backend/services.py`）
3. 修复并补齐基础测试可用性
4. 清理仓库运行时产物并完善忽略规则
5. 对齐项目文档与实际实现

## 执行顺序与验收标准

### 1) 密钥安全整改（P0）
- 范围：`backend/test.py` 及相关配置文档
- 动作：
  - 移除硬编码密钥
  - 改为环境变量读取（未配置时给出明确提示）
- 验收：
  - 仓库中不再出现明文 API Key
  - 示例代码不会在默认情况下泄露敏感信息

### 2) 服务层代码收敛（P0/P1）
- 范围：`backend/services.py`
- 动作：
  - 删除无效/重复 import
  - 删除已废弃的注释代码块
  - 修复状态结构不一致问题，避免运行期隐患
- 验收：
  - 文件可读性提升
  - 无明显重复/死代码
  - 相关接口行为保持兼容

### 3) 测试修复（P1）
- 范围：`backend/test_smoke.py`
- 动作：
  - 修复与当前函数签名不一致的测试
  - 修复与当前绘图行为不一致的断言
- 验收：
  - `python -m unittest test_smoke.py` 通过

### 4) 仓库卫生与忽略规则（P1）
- 范围：`.gitignore`
- 动作：
  - 补充虚拟环境目录与运行时文件忽略规则（如 `backend/venv311/`）
- 验收：
  - 常见本地产物不再持续污染工作区

### 5) 文档对齐（P1）
- 范围：`PROJECT_OVERVIEW.md`（必要时同步 `README.md`）
- 动作：
  - 删除与现状不符的接口描述（如 `/api/v1/...`）
  - 改为与当前 FastAPI 实际接口一致
- 验收：
  - 文档可直接指导当前代码版本使用
